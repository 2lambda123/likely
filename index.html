Literate Kernel Library

Beautiful image processing functions for heterogenous architectures.

Introduction

Welcome to the documentation and source code for Likely the literate kernel library! Simply stated, the goals of Likely are to provide image processing and machine learning functions, or kernels, that are:

  • executable everywhere.
  • perfectly optimized.
  • uncompromisingly elegant.

Executable Everywhere

Likely is written in plain old ISO C++ '98 and depends only on LLVM which means it runs everywhere LLVM does. In addition to executing natively on a single host CPU thread via the LLVM Just-In-Time (JIT) compiler, kernels can also target multi-core CPUs using OpenMP and GPUs and other OpenCL compatible coprocessors.

Perfectly Optimized

Likely relies on powerful and well established LLVM optimization passes to make kernel execution as fast as possible. Since compilation is held off until runtime, kernels can be optimized to take advantage of all instruction set extensions available on the host processor. In reality, compilation is delayed until a function is called for the first time, enabling optimization opportunities that are impossible in conventional image processing libraries. In fact the entire premise of Likely is hinged on the simple yet powerful observation that for image processing applications

while kernels must be written generically to handle any matrix type, at runtime they tend to be executed repeatedly on the same type.

The repeated execution of a kernel with the same matrix type means that no branching is needed to handle different data types, entire loops and code blocks can be eliminated, and many values that would be runtime parameters instead become compile time constants. For this reason, it's not uncommon for Likely kernels to be faster than equally generic C code.

In Likely, a matrix's type is referred to as its hash, which encompasses its data type (integer or floating point), depth (8, 16, 32, or 64 bits), where the computation involving it should occur (single-threaded CPU, OpenMP CPU, or OpenCL coprocessor), and which of its axes are single-dimension. Likely functions check the input's hash against the currently compiled kernel's hash and recompile on a mismatch. In this way,

all the logic that is generally needed to handle different matrix types is reduced to a single 16-bit integer comparison which branch prediction can optimize out.

Uncompromisingly Elegant

Likely is inspired by Donald Knuth's literate programming. It is a literate kernel library because

all the kernels documented on this webpage are also the source code for the library.

In fact, this webpage is compiled into the Likely library and parsed at runtime to extract kernel definitions. You can access this content for yourself with const char *this_webpage = likely_index_html();.

Kernels are written in MathML which allows them to be rendered cleanly here and parsed easily by the backend.
