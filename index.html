Literate Kernel Library

Beautiful image processing functions for heterogenous architectures.

Introduction

Welcome to the documentation and source code for Likely the literate kernel library! Simply stated, the goals of Likely are to provide image processing and machine learning functions, or kernels, that are:

  • executable everywhere.
  • perfectly optimized.
  • uncompromisingly elegant.

Executable Everywhere

Likely is written in plain old ISO C++ '98 and only depends on LLVM which means it runs everywhere LLVM does. In addition to executing natively on a single host CPU thread via the LLVM Just-In-Time (JIT) compiler, kernels can also target multi-core CPUs using OpenMP and GPUs and other OpenCL compatible coprocessors.

Perfectly Optimized

Likely relies on the powerful and well established collection of LLVM optimization passes to make kernel execution as fast as possible. Since compilation is held off until runtime, kernels can be optimized to take advantage of all instruction set extensions available on a processor. Indeed compilation is delayed until a function is called for the first time, which means that Likely can make optimizations that would be impossible in a conventional image processing library. No branching is needed to handle different data types, entire loops and code blocks can be eliminated as an input's dimensions and type is known, and many values that would be runtime parameters become compile time constants.

These optimizations are made possible by a simple but powerful observation for image processing applications:

While kernels must be defined generically to handle any matrix type, at runtime they tend to be executed repeatedly on the same type.
